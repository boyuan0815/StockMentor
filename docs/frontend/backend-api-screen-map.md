# Backend API Screen Map

This file maps frontend screens to the current Spring Boot backend API surface. Future frontend implementation chats
should use this file instead of guessing endpoint names.

Common conventions:

- Normal user endpoints require HTTP Basic Auth unless marked public.
- Admin endpoints require HTTP Basic Auth with `ROLE_ADMIN` and `X-Admin-Token`.
- Current-user user actions must not send `userId`; the backend resolves the user.
- Errors are normalized on the frontend from backend status and message fields where available.
- Backend security failures under `/api/**` return JSON 401/403 bodies; Expo Web CORS origins are controlled by
  `stockmentor.cors.allowed-origins`, with `OPTIONS /**` preflight permitted.
- US008 is admin stock market data maintenance.
- US009 is beginner stock market data viewing.
- US010 is simulated paper trading.
- US011 is backend/system automatic stock retrieval.
- US012 is user-facing AI stock explanation.

## Delayed Market Data Contract

The intended product behavior is 15-minute delayed educational market data:

```text
displayedMarketTime = current New York market time - 15 minutes
```

This formula applies during the active delayed display window. During pre-open, post-close, weekend, or holiday
behavior, the backend may select the latest completed trading day or stored daily close metadata instead.

Current backend contract: stock list/detail/history responses expose delayed display fields such as `displayedPrice`,
`displayedAbsoluteChange`, `displayedPercentChange`, `previousClose`, `displayedMarketTime`,
`targetDisplayMarketTime`, `dataDelayMinutes`, `priceFreshnessStatus`, `priceFreshnessLabel`, `isPriceAvailable`,
`isTradeExecutable`, `dataNote`, `priceSource`, `marketTimeZone`, and where applicable `lastBackendUpdatedAt`.
Watchlist stock rows expose the same delayed display fields while retaining legacy `currentPrice` and `percentChange`
compatibility fields.

Frontend display should prefer backend delayed fields. Legacy `currentPrice`, `percentChange`, and `lastUpdated` remain
backward-compatible stock table snapshot fields. Stock detail `dataSource` is a legacy analysis-source field;
`analysisDataSource`, `snapshotHighPrice`, `snapshotLowPrice`, and `snapshotTimeframe` describe the latest analysis
snapshot. `priceSource` is the authoritative displayed quote source.

Stock detail may expose `previousClose`, `displayedAbsoluteChange`, and `displayedVolume` for the Phase 3B quote panel.
Use them when present and leave values hidden when null. Watchlist rows expose the same delayed display fields as stock
list/detail rows while retaining legacy compatibility fields.

Backend movement is previous-close based: `displayedAbsoluteChange = displayedPrice - previousClose` and
`displayedPercentChange = displayedAbsoluteChange / previousClose * 100`. Canonical freshness labels are `Delayed 15 min`,
`Market Closed · Last Close`, `Latest Stored Price`, and `Unavailable`; frontend should display the label instead of
duplicating market-time logic.

## Phase 3B Mobile Route Standard

Visible beginner tabs:

- `Watchlist`
- `Stocks`
- `Suggestions`
- `Portfolio`
- `Profile`
- `Search`

Hidden tab routes should use `href: null`, including stock detail, contextual search, stock explanation compatibility
route, paper trade tickets, and transaction/detail routes.

Current stock UI routing:

- Watchlist tab opens the `Watchlists` table page.
- Stocks tab opens the compact `Stocks` table page.
- Search tab is the standalone rightmost search route with a visual separator.
- Contextual search uses hidden `/stocks/search-context`.
- Detail rows pass explicit return params: `stocks`, `watchlist`, `search-context`, or `search-tab`.
- Paper-trade navigation preserves return params. Buy/sell APIs are called only from stock-scoped guarded tickets after
  whole-share quantity input and explicit confirmation.
- `/paper-trading/transactions` opens or reuses the Portfolio `History` tab.
- Portfolio bottom-tab entry opens `Assets` at the top; `/paper-trading?tab=history` intentionally opens History.

Current stock UI surfaces:

- Watchlist table columns: `No.`, `Symbol`, `Price`, `Chg %`. Sortable: `Symbol`, `Price`, `Chg %`.
- Stocks table columns: `No.`, `Symbol`, `Price`, `Chg %`, `Action`.
- Search empty input shows search history and max three `Latest Viewed Stocks`; it does not show the full supported
  stock list.
- Typed search results show symbol/name plus a heart toggle and no price/change.
- Stock detail AI drawer titles are `View AI Stock Explanation` and `Close AI Stock Explanation`; backend cache/generated
  status messages are hidden from the user.

## DTO Reference

- `RegisterRequest`: `email`, `username`, `password`, `confirmPassword`.
- `AuthUserResponse`: `userId`, `email`, `username`, `role`, `status`, `onboardingCompleted`, `hasInvestmentProfile`,
  `mustCompleteOnboarding`, `createdAt`, `lastLoginAt`.
- `OnboardingSubmitRequest`: `answers`.
- `PaperTradeRequest`: `symbol`, `quantity`.
- `AdminUpdateUserStatusRequest`: `status`.
- `AdminBackfillRequest`: `type`, `symbols`, `date`, `startDate`, `endDate`.

For full field lists, mirror the Java records in `backend/src/main/java/net/boyuan/stockmentor/**/dto`.
